# Bezpieczeństwo i prywatność

## Nie publikuj

- danych osobowych i kontaktowych;
- haseł, tokenów, kluczy API i plików `.env`;
- dokumentów oznaczonych jako poufne lub wewnętrzne;
- danych technicznych, handlowych i operacyjnych bez zgody;
- treści objętych NDA, prawem autorskim lub tajemnicą przedsiębiorstwa.

## Dobra praktyka

- anonimizuj nazwy osób, firm, lokalizacji i urządzeń;
- używaj fikcyjnych danych w przykładach;
- podawaj AI tylko minimalny potrzebny zakres informacji;
- sprawdzaj regulamin narzędzia AI i politykę organizacji;
- traktuj odpowiedzi AI jako szkic wymagający kontroli człowieka;
- ustaw repozytorium jako prywatne, jeśli zawiera materiały robocze.

`.gitignore` pomaga ograniczyć przypadkowe dodanie typowych sekretów, ale nie zastępuje ostrożności.
